# Política de seguridad

## Versiones compatibles

Mientras el proyecto esté antes de la versión 1.0, sólo la rama `main` recibe correcciones de seguridad.

## Informar una vulnerabilidad

No publique vulnerabilidades ni credenciales en un issue. Use **Security > Report a vulnerability** en GitHub para enviar un reporte privado. Incluya el impacto, los pasos mínimos para reproducirlo y una mitigación sugerida si la conoce.

Los mantenedores intentarán confirmar la recepción en un plazo de 7 días y coordinarán la divulgación después de disponer de una corrección. Nunca adjunte imágenes de usuarios, claves de API, archivos `.env` ni volcados de base de datos.

Si el alojamiento todavía no tiene habilitados los avisos privados de seguridad, contacte de forma privada al propietario del repositorio desde su perfil.
